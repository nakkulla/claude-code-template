# Claude Code 글로벌 지침

이 문서는 Claude Code 실행 시 모든 프로젝트에 적용되는 기본 지침을 담고 있습니다.

---

## 1. 문서화 규칙

### 계획과 결과 문서화
- 작업을 계획하게 되면 반드시 문서화하며, 해당 작업 폴더 내부에 `docs` 폴더를 생성하여 계획과 결과 파일을 둡니다.
- 문서 이름 형식:
  - 계획: `plan-{YYYYMMDD-HHMM}.md`
  - 결과: `result-{YYYYMMDD-HHMM}.md`
- 날짜와 시간은 24시간 기준으로 작업을 시작하거나 결과를 확보한 시점을 사용합니다.

### 언어 설정
- 문서는 모두 한국어로 작성합니다.
- 참고한 템플릿이나 폴더 구조 문서를 링크하거나 언급하여 맥락을 남깁니다.

---

## 2. 코드 작성 원칙

### 스타일
- 간결하고 읽기 쉬운 코드를 작성합니다.
- 불필요한 주석은 피하고, 코드 자체가 의미를 전달하도록 합니다.
- 함수와 변수명은 명확하게 작성합니다.

### 테스트
- 중요한 로직에는 테스트를 작성합니다.
- 변경사항 적용 전 기존 테스트를 확인합니다.

---

## 3. Git 규칙

### 커밋 메시지
- 명확하고 간결한 커밋 메시지를 작성합니다.
- 접두사 사용: `Add:`, `Update:`, `Fix:`, `Remove:`

### 브랜치
- 기능 개발은 별도 브랜치에서 진행합니다.
- 완료 후 main/master에 병합합니다.

---
